# Guía de contribución técnica

Esta guía es para quienes quieren **editar el wiki directamente con Git**: hacer fork, crear ramas y abrir pull requests. Si prefieres simplemente enviar contenido, comentarios o correcciones por correo (sin conocimientos técnicos), consulta **[Cómo Contribuir](/es/start/contribute)**.

Así funciona el flujo de edición directa:

## Pasos para contribuir

1. **Haz un fork o clona el repositorio** (si corresponde).  
2. **Crea una rama** con un nombre como `tema-tunombre` o similar.  
3. Ve a la carpeta de la sección que quieras editar o donde quieras agregar contenido (por ejemplo, `/glossary`).  
4. Crea o edita archivos `.md`.  
   - Usa el estilo y el frontmatter existentes (el bloque `---` al inicio).  
   - Mantén los párrafos cortos y claros.  
   - Agrega enlaces a páginas relacionadas de este wiki.  
   - Usa lenguaje sencillo; explica los términos que puedan resultar poco familiares.  
   - Incluye citas cuando hagas referencia a hechos, datos o políticas.  
5. **Previsualiza** tus cambios (en tu editor local o mediante la vista previa del wiki) para revisar el formato y la accesibilidad (por ejemplo, texto alternativo para las imágenes, estructura de encabezados).  
6. **Envía un pull request** / merge request / actualización de contenido (según tu flujo de trabajo). En la descripción incluye:  
   - Qué cambió y por qué.  
   - Qué sección o secciones podrían necesitar revisión o seguimiento.  
   - Cualquier consideración de accesibilidad o pregunta sin resolver.  
7. **Proceso de revisión**  
   - Una persona moderadora o coeditora revisará tu contribución.  
   - Es posible que te pidan aclaraciones o ediciones.  
   - Una vez aprobada, tus cambios se fusionarán y se publicarán.

## Pautas para quienes contribuyen

- Escribe en **lenguaje sencillo**: las oraciones cortas y los términos claros ayudan a más personas.  
- Ten en cuenta la **navegación accesible**: encabezados, listas, enlaces y texto alternativo para cualquier elemento visual.  
- Respeta el principio **"Nada Sobre Nosotros Sin Nosotros"**: cuando escribas sobre una comunidad con discapacidad, pon en el centro las voces de esa comunidad y evita suposiciones.  
- Evita la jerga cuando sea posible; cuando debas usarla, enlaza a un término del glosario o explícalo.  
- Usa lenguaje inclusivo: revisa tu tono, los pronombres y la representación.

## Cómo obtener apoyo

- **Escríbenos:** [contribute2disabilitywiki@gmail.com](mailto:contribute2disabilitywiki@gmail.com) — son bienvenidas tus preguntas, comentarios o contribuciones de cualquier tipo, y no necesitas conocimientos técnicos para escribirnos.
- Consulta estas páginas para obtener ayuda:  
  - [Terminología](/es/glossary/terminology) — para definiciones de términos.  
  - [Pautas editoriales](/es/glossary/editorial-guidelines) — para el estilo y el proceso.  
- Si no estás seguro de un tema o de cómo expresarlo, déjalo como borrador y pide una revisión entre pares.

¡Gracias por ayudar a construir este recurso! Tu voz importa, especialmente si tienes experiencia vivida.
